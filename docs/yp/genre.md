# YP4G 拡張ジャンル文字列フォーマット仕様

PeerCast の `PCP_CHAN_INFO_GENRE`（タグ `gnre`）は任意のジャンルテキストだが、
YP サーバ向けに構造化された書式を持ち、YP の挙動を制御できる。

## 書式

```
{YPプレフィックス}[{ネームスペース}:][?][@...]ジャンル文字列
```

| 要素 | 説明 | 例 |
|---|---|---|
| YP プレフィックス | YP 識別子（このYPでは `yp` 固定） | `yp` |
| ネームスペース | 省略可。英数字 + `:` で終端 | `ABC:` |
| `?` | リスナー数非表示フラグ | `?` |
| `@` | ポートチェック有効 | `@` |
| `@@` | ポートチェック + 帯域チェック（配信ビットレート基準） | `@@` |
| `@@@` | ポートチェック + 帯域チェック（2Mbps 固定制限） | `@@@` |
| ジャンル文字列 | 任意のジャンルテキスト | `ゲーム` |

例（完全形）: `ypABC:?@@ゲーム` → プレフィックス=yp, NS=ABC, 非表示=true, 帯域チェック, ジャンル=ゲーム

---

## 本実装のスコープ

| 機能 | 実装 |
|---|---|
| YP プレフィックス | **`yp` 固定**（不一致はこの YP 向けでないとして無視） |
| ネームスペース | **無視**（存在しても読み飛ばす） |
| `?` リスナー非表示 | **実装済み** |
| `@` 帯域制限フラグ | **無視**（存在しても読み飛ばす） |
| ジャンル文字列 | **実装済み** |

### 処理する書式

```
yp[ネームスペース:][?][@@@]ジャンル文字列
```

- `yp` で始まらないチャンネルはこの YP 向けでないとして無視する
- `?` がある場合、`index.txt` のリスナー数・リレー数を `-1` で出力する
- ジャンル文字列のみを抽出して保存・表示する

### パース例

| ジャンル文字列 | 対象YP | 非表示 | ジャンル |
|---|---|---|---|
| `ypゲーム` | ✓ | false | `ゲーム` |
| `yp?ゲーム` | ✓ | true | `ゲーム` |
| `ypABC:?@@ゲーム` | ✓ | true | `ゲーム` |
| `spゲーム` | ✗（別YP） | — | — |
| `ゲーム` | ✗（プレフィックスなし） | — | — |
