# ジャンル文字列フォーマット仕様

PeerCastの `PCP_CHAN_INFO_GENRE`（タグ `gnre`）は任意のジャンルテキストだが、
YPサーバ向けに構造化された書式を持ち、YPの挙動を制御できる。

出典: `_ref/peercast-yayp/docs/worker.md`
パーサ参考実装: `_ref/peercast-yayp/internal/peercast/genre.go`

---

## 完全な書式（参考）

```
{YPプレフィックス}[{ネームスペース}:][?][@...]{ジャンル文字列}
```

| 要素 | 説明 | 例 |
|---|---|---|
| YPプレフィックス | YP識別子（このYPでは `yp` 固定） | `yp` |
| ネームスペース | 省略可。英数字 + `:` 区切り | `ns:` |
| `?` | リスナー数非表示フラグ | `?` |
| `@` | 帯域制限（`@`=FW、`@@`=BW、`@@@`=HBW） | `@@` |
| ジャンル文字列 | 任意のジャンルテキスト | `ゲーム` |

例（完全形）: `ypns:?@@ゲーム` → プレフィックス=yp, NS=ns, 非表示=true, 制限=BW, ジャンル=ゲーム

---

## 本実装のスコープ

| 機能 | 実装 |
|---|---|
| YPプレフィックス | **`yp` 固定** |
| ネームスペース | **実装しない**（存在しても無視） |
| `?` リスナー非表示 | **実装する** |
| `@` 帯域制限 | **実装しない**（存在しても無視） |
| ジャンル文字列 | **実装する** |

### 本実装が処理する書式

```
yp[?]{ジャンル文字列}
```

- `yp` で始まらないチャンネルはこのYPに登録されていないものとして無視する
- `?` がある場合、`index.txt` のリスナー数・リレー数を `-1` で出力する
- その後の文字列をジャンルとして使用する

### パース例

| ジャンル文字列 | 対象YP | 非表示 | ジャンル |
|---|---|---|---|
| `ypゲーム` | ✓ | false | `ゲーム` |
| `yp?ゲーム` | ✓ | true | `ゲーム` |
| `ap?ゲーム` | ✗（別YP） | — | — |
| `ゲーム` | ✗（プレフィックスなし） | — | — |

## パーサ実装メモ

参考実装（`genre.go`）の処理順序：

1. プレフィックス一致確認 → 不一致なら `ok=false`
2. `:` が出現した位置までをネームスペースとして記録（本実装では読み飛ばす）
3. `?` → `HiddenListeners = true`
4. `@` → 帯域制限カウントアップ（本実装では読み飛ばす）
5. 英数字以外の文字が出現した時点でフラグパース終了
6. 残りをジャンル文字列として返す

> **注意**: ネームスペース・帯域制限フラグを無視する場合でも、
> それらが含まれる文字列を誤ってジャンル文字列に混入させないよう、
> 正しく読み飛ばす処理が必要。
